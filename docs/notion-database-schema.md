# Notion Database Schemas

All three databases should be created in the same Notion parent page or
workspace. JSON schema files for each are in `config/notion-databases/`.

---

## Database 1: Job Tracker

**Purpose:** Stores every job saved via the Chrome extension. The central
hub for Phase 2 — Claude reads from and writes back to this database.

| Property Name | Type | Notes |
|---|---|---|
| Role Title | Title | Default title property |
| Company | Text | |
| Job URL | URL | Link to the original posting |
| Raw JD | Text | Full job description text — required for AI review |
| Status | Select | See options below |
| AI Notes | Text | Written by Claude: fit assessment, resume selection rationale |
| Resume | Text | Full optimized resume text written by Claude |
| Location | Text | |
| Job Type | Select | Remote, Hybrid, Onsite |
| Salary Range | Text | |
| Date Added | Date | |
| Date Applied | Date | |
| ATS Source | Text | e.g. Greenhouse, Lever, Workday |
| Notes | Text | Your personal notes |

**Status options:**
- Bookmarked — saved, not yet reviewed
- AI Reviewed — Claude has processed and written back
- AI Reviewed - Skip — Claude flagged as poor fit
- Applying — you've decided to apply
- Applied — application submitted
- Interviewing — active interview process
- Offer — offer received
- Closed — rejected, withdrew, or role filled

---

## Database 2: Resume Repository

**Purpose:** Stores your base resume variants and the master Resume Selection
Rules document. Claude reads from this database during every job review.

| Property Name | Type | Notes |
|---|---|---|
| Resume Name | Title | e.g. "Director of Engineering - Base" |
| Category | Select | Director, Senior Manager, VP, AI/Technical Consulting, Other |
| Version | Text | e.g. "v1.0", "v1.1" |
| Last Updated | Date | |
| Resume Content | Text | Full plain text of the resume |
| Active | Select | Yes / No |
| Notes | Text | e.g. "Use for roles reporting to CTO" |

**Special rows (non-resume):**
- `Resume Selection Rules v1.0` — Category: Other, contains the full
  configuration document generated during onboarding. Claude reads this
  on every job review to apply your preferences.

---

## Database 3: Interview Prep Hub

**Purpose:** One row per opportunity being actively pursued. Stores all
prep materials generated by Claude and links back to the Job Tracker row
and the Claude Project for that opportunity.

| Property Name | Type | Notes |
|---|---|---|
| Opportunity | Title | e.g. "Director of Engineering @ Stripe" |
| Company | Text | |
| Role Title | Text | |
| Status | Select | See options below |
| Claude Project URL | URL | Link to the Claude Project for this opportunity |
| Job Tracker URL | URL | Link to the corresponding Job Tracker row |
| Resume Used | Text | Name of the resume variant used |
| Interview Round | Select | See options below |
| Interview Date | Date | |
| Cheat Sheet | Text | Generated by Claude — stories, talking points, questions |
| Company Research | Text | Generated by Claude — company overview, recent news |
| Key Talking Points | Text | Top 3-5 things to emphasize for this role |
| Questions To Ask | Text | Smart questions generated for this specific role |
| Mock Interview Notes | Text | Notes from Claude-run mock interview sessions |
| Post Interview Notes | Text | Your debrief after each round |
| AI Coaching Notes | Text | Claude's feedback on your mock interview answers |

**Status options:**
Active Prep, Interview Scheduled, Interview Complete, Offer, Closed

**Interview Round options:**
Phone Screen, Recruiter Screen, Hiring Manager, Panel, Final Round, Offer Stage
