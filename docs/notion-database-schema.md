# Notion Database Schemas

All three databases should be created in the same Notion parent page or
workspace. JSON schema files for each are in `config/notion-databases/`.

---

## Database 1: Job Tracker

**Purpose:** Stores every job saved via the Chrome extension. The central
hub for Phase 2 — Claude reads from and writes back to this database.

| Property Name | Type | Written by | Notes |
| --- | --- | --- | --- |
| Role Title | Title | Extension | Default title property |
| Company | Text | Extension | |
| Job URL | URL | Extension | Link to the original posting |
| Raw JD | Text | Extension | Full job description text — required for AI review |
| Status | Select | Claude / User | Claude sets AI Reviewed or AI Reviewed - Skip; user sets the rest |
| AI Notes | Text | Claude | Fit tier, resume selection rationale, strengths, gaps, recommendation |
| Resume | Text | Claude | Full optimized resume text for this specific role |
| Location | Text | Extension | |
| Job Type | Select | Extension | Remote, Hybrid, Onsite |
| Salary Range | Text | Extension | |
| Date Added | Date | Extension | |
| Date Applied | Date | User | |
| ATS Source | Text | Extension | e.g. Greenhouse, Lever, Workday |
| Notes | Text | User | Personal notes — not read by Claude |

**Status options:**

- Bookmarked — saved, not yet reviewed
- AI Reviewed — Claude has processed and written back
- AI Reviewed - Skip — Claude flagged as poor fit
- Applying — you've decided to apply
- Applied — application submitted
- Interviewing — active interview process
- Offer — offer received
- Closed — rejected, withdrew, or role filled

## Database 2: Resume Repository

**Purpose:** Stores your base resume variants and the master Resume Selection
Rules document. Claude reads from this database during every job review.

| Property Name | Type | Written by | Notes |
| --- | --- | --- | --- |
| Resume Name | Title | Claude (onboarding) | e.g. "Director of Engineering - Base" |
| Category | Select | Claude (onboarding) | Director, Senior Manager, VP, AI/Technical Consulting, Other |
| Version | Text | Claude (onboarding) | e.g. "v1.0", "v1.1" |
| Last Updated | Date | Claude (onboarding) | Set to today's date on creation |
| Resume Content | Text | Claude (onboarding) | Full plain text of the resume or rules document |
| Active | Select | Claude / User | Yes / No — Claude only uses rows marked Yes |
| Notes | Text | User | e.g. "Use for roles reporting to CTO" |

**Special rows (non-resume):**

- `Resume Selection Rules v1.0` — Category: Other, contains the full
  configuration document generated during onboarding. Claude reads this
  on every job review to apply your preferences.

## Database 3: Interview Prep Hub

**Purpose:** One row per opportunity being actively pursued. Stores all
prep materials generated by Claude and links back to the Job Tracker row.

| Property Name | Type | Written by | Notes |
| --- | --- | --- | --- |
| Opportunity | Title | Claude (init) | Format: "[Role Title] @ [Company]" |
| Company | Text | Claude (init) | Copied from Job Tracker row |
| Role Title | Text | Claude (init) | Copied from Job Tracker row |
| Status | Select | Claude / User | Claude sets Active Prep on creation; user updates the rest |
| Job Tracker URL | URL | Claude (init) | Link back to the Job Tracker row for this role |
| Resume Used | Text | Claude (init) | Name of the resume variant selected in Phase 2 |
| Interview Round | Select | User | Current or upcoming round |
| Interview Date | Date | User | Scheduled interview date |
| Cheat Sheet | Text | Claude (init/update) | Living document: self-intro, why this company, accomplishments, STAR stories, talking points, gaps, questions, comp notes |
| Company Research | Text | Claude (init) | Business overview, recent news, engineering culture, role context |
| Key Talking Points | Text | Claude (skill) | Top 3-5 things to emphasize — extracted from Cheat Sheet |
| Questions To Ask | Text | Claude (skill) | 5 smart, company-specific questions — extracted from Cheat Sheet |
| Mock Interview Notes | Text | Claude (mock) | Saved after each mock interview session |
| Post Interview Notes | Text | Claude (debrief/prep) | Real interview debriefs and interviewer-specific cheat sheets, appended with headers |
| AI Coaching Notes | Text | Claude (mock) | Structured feedback on mock interview performance |

**Status options:**

- Active Prep — set by Claude when row is created
- Interview Scheduled — user updates when interview is confirmed
- Interview Complete — user updates after interview
- Offer — offer received
- Closed — rejected, withdrew, or role filled

**Interview Round options:**
Phone Screen, Recruiter Screen, Hiring Manager, Panel, Final Round, Offer Stage
